# Adjudicator Overview

## Your Role

As an **Adjudicator**, you resolve complex cases, edge scenarios, and reviewer disagreements. You're the final decision-maker who ensures consistency and interprets guidelines when situations aren't clear-cut.

## Key Responsibilities

- ⚖️ **Resolve Disputes** - Make final decisions on reviewer disagreements
- 🔍 **Handle Edge Cases** - Evaluate submissions that don't fit standard patterns
- 📋 **Interpret Guidelines** - Apply guidelines to novel situations
- 📊 **Ensure Consistency** - Set precedents for future similar cases
- 🎓 **Educate** - Help reviewers understand complex decisions

## When Cases Come to You

Submissions reach adjudication when:

1. **Reviewer Uncertainty** - Reviewer can't confidently decide
2. **Reviewer Disagreement** - Multiple reviewers have conflicting opinions
3. **Novel Scenarios** - Situation not covered by current guidelines
4. **Appeal** - Submitter contests a rejection
5. **Quality Audit** - Random sampling for calibration

## Your Decision Framework

### 1. Review the Context
- What's the specific question or disagreement?
- What have reviewers already said?
- What guidelines apply?

### 2. Evaluate Against Standards
- Does it meet the core quality bar?
- Is it useful for model training?
- Does it align with project goals?

### 3. Consider Precedent
- Have we seen similar cases?
- What decisions were made before?
- Should this set a new precedent?

### 4. Make a Clear Decision
- Provide definitive ruling (accept/reject)
- Explain reasoning thoroughly
- Note if this creates a new precedent

## Decision Principles

:::tip Core Principles
1. **Consistency First** - Similar cases should have similar outcomes
2. **Intent Over Form** - Focus on whether submission achieves its purpose
3. **Training Value** - Would this improve model performance?
4. **Clarity** - When in doubt, favor clear over clever
:::

## Impact of Your Decisions

Your rulings:
- Set precedents for future cases
- Guide reviewer calibration
- Shape interpretation of guidelines
- Influence project direction
