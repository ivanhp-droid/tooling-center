# Submitter Checklist

Use this checklist for every submission to ensure quality and consistency.

## Pre-Submission

### Step 1: Intent Definition
- [ ] Intent clearly states what the AI should accomplish
- [ ] Success criteria are measurable and unambiguous
- [ ] Intent is scoped appropriately (not too broad, not too narrow)
- [ ] Intent aligns with project failure mode categories

### Step 2: System Prompt
- [ ] System prompt follows template structure
- [ ] Instructions are clear and complete
- [ ] No contradictory rules or guidelines
- [ ] Appropriate tone and persona defined
- [ ] Tool usage permissions specified if needed

### Step 3: User Prompts
- [ ] User prompts naturally test the intent
- [ ] Prompts are realistic and well-formed
- [ ] Sufficient context provided for the AI
- [ ] Edge cases or complexities included where appropriate

### Step 4: Bad Response
- [ ] Demonstrates a specific, identifiable failure mode
- [ ] Failure is clear and obvious (not subtle)
- [ ] Response is realistic (could plausibly come from an AI)
- [ ] Failure mode documented in response notes

### Step 5: Good Response
- [ ] Directly addresses the user prompt
- [ ] Follows system prompt instructions
- [ ] Avoids the failure mode shown in bad response
- [ ] Demonstrates best practices
- [ ] Natural and human-like in tone

## Final Review

- [ ] All required fields completed
- [ ] Spelling and grammar checked
- [ ] Consistent formatting throughout
- [ ] Links or references verified (if applicable)
- [ ] Reviewed against [Common Mistakes](common-mistakes)
