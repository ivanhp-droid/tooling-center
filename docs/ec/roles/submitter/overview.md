# Submitter Overview

## Your Role

As a **Submitter**, you are responsible for creating high-quality task submissions that demonstrate proper AI behavior and failure mode examples. Your work forms the foundation of the training data used to improve AI models.

## Key Responsibilities

- 📝 Craft clear, well-structured intents and prompts
- 🤖 Generate appropriate system prompts that align with project goals
- ✅ Create "good" responses that demonstrate ideal AI behavior
- ❌ Create "bad" responses that illustrate specific failure modes
- 🔍 Ensure all submissions meet quality standards before review

## Success Criteria

Your submissions should:

1. **Be Clear and Unambiguous** - Intents and prompts should have a single, obvious interpretation
2. **Follow Template Structures** - Use established patterns for system prompts and responses
3. **Demonstrate Real Failure Modes** - Bad responses should represent actual problematic behaviors
4. **Include Complete Context** - Provide all necessary information for reviewers

## What Makes a Good Submission?

:::tip Key Traits
- Clear intent that defines success criteria
- System prompt that enables the desired behavior
- User prompts that test the AI appropriately
- Bad response that demonstrates a specific failure mode
- Good response that shows the correct behavior
:::
